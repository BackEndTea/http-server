Why Do I Need a PHP Server?

Aerys cuts out the middle man and gives PHP apps full access to the HTTP protocol

PHP earned its niche in the web development community by making HTTP a first-class citizen of the language. If you're thinking that PHP is already a web framework, you'd be right. So why should we reinvent the wheel and write a web server in PHP? Well ...

Why? For the Love of Lerdorf, Why?

  • PHP's process-per-request SAPI model doesn't scale well
  • PHP's Web SAPIs can't support the large number of concurrent client connections needed for websockets
  • PHP's dependence on third party webservers prevent websocket applications without an additional proxy server
  • PHP's HTTP abstractions are leaky and don't support advanced HTTP features like Expect: 100-continue headers
  • PHP's shared-nothing SAPI architecture makes opcode caching a requirement
  • PHP has superglobals (gag)
  • Third-party webservers mean you can't easily tailer the server to fit the needs of your application

How Does Aerys Solve These Problems?

  • Aerys is a single-process event-driven server capable of handling tens of thousands of concurrent connections at once
  • Aerys integrates full websocket support as a first-class citizen of the server
  • Aerys doesn't need a middle man; it's your web server and application rolled into one
  • Because Aerys handles all requests in a single process there is no need for opcode caching
  • Aerys provides a simple modding interface to allow app-specific customization and resource caching
  • Aerys gives applications full access to all aspects of the HTTP/1.1 protocol
  • It's really fast; as fast or faster than PHP apps behind Apache/Nginx/etc even if those apps use opcode caching

What are the Drawbacks?

  • Memory management requires a more hands on approach since each request isn't killed off when the process completes
  • Some form of multiprocessing is necessary to prevent blocking application IO
  • Error tolerance is lower; fatal application errors can bring down your entire server
  • Developers need slightly more knowledge of the HTTP protocol than with standard PHP.

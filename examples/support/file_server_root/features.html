So What Does it Do?

Besides make your wildest dreams come true ...

  • Handles tens of thousands of concurrent clients without a hiccup
  • Serves static files ... FAST!
  • Serves PHP applications ... FAST! Faster than standard web SAPIs even if they utilize opcode caching
  • Offers full websocket support as a first class citizen
  • Supports TLS encryption for both https:// and wss:// resources to secure your data
  • Serves websocket endpoints alongside PHP applications without the need for proxy servers
  • Offers built-in mods for request logging, rate-limiting, etc
  • Exposes request lifecycle event hooks for app-specific caching solutions (or anything else)
  • Eliminates the need for 3rd-party software like apache/nginx/lighttpd/etc

What It Doesn't Do

PHP is traditionally a web framework above all else. But Aerys treats PHP like the programming language that it is -- not like a web framework. This means that Aerys does not (because it's a server, not a framework) do some of the HTTP massaging that PHP users have come to expect:

  • Pre-parse cookies and session data for you
  • Provide superglobals a-la $_GET and $_POST
  • etc.

Instead, Aerys provides applications with all of the information about the request using a standard, CGI-style array. From here frameworks and applications can easily use Aerys's modular architecture to supply any necessary functionality. In short, Aerys treats PHP developers like real programmers and not the morons of the web development world. In return, developers get full control over their HTTP experience and better performance/scalability than what would otherwise be possible with PHP.

Websocket OMG can PHP really solve the C10K Problem and do webscale? Yes. Aerys makes websockets as outlined in RFC 6455 a first class citizen. Traditional PHP websocket solutions require a proxy server because the real front-facing server -- be it Lighttpd, Apache, Nginx, etc -- is already taking up the ports (80/443) on which the websocket protocol listens. Because Aerys is its own event-driven server it can delegate control of the appropriate client sockets as needed to its own websocket handling mechanism.
